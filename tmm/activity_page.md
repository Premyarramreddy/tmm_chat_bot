# Activity Module – RAG Knowledge Document

## 1. Overview

The Activity Module allows users to manage and track all profile interactions, including requests, favourites, likes, views, and account actions such as blocking and reporting.

Users can access the Activity page through the footer navigation.

---

## 2. Entry Point

* Location: Footer → Activity Icon
* Access Condition: User must be logged in

---

## 3. Activity Page Structure

The Activity page contains multiple tabs. Each tab displays a count of related profiles.

All data is updated in real time.

---

## 4. Tabs and Features

### 4.1 Match Requests

Purpose: View profiles that have sent connection requests.

Features:

* Accept request
* Reject request
* View request count

---

### 4.2 Favourites

Purpose: View profiles that have added the user to their favourites list.

Features:

* View favourited profiles
* View count

---

### 4.3 Super Likes

Purpose: View profiles that have liked the user’s profile.

Features:

* View liked profiles
* View count

---

### 4.4 Accepted

Sub-tabs:

* Accepted by Me
* Accepted by Others

Purpose: View accepted connection requests.

Features:

* View accepted profiles
* Filter by acceptance type

---

### 4.5 Sent

Purpose: View profiles to whom connection requests have been sent and are pending.

Features:

* View pending requests
* Track status

---

### 4.6 Rejected

Sub-tabs:

* Rejected by Me
* Rejected by Others

Purpose: View rejected connection requests.

Features:

* View rejected profiles
* Filter by rejection type

---

### 4.7 Views

Sub-tabs:

* Viewed by Me
* Viewed by Others

Purpose: Track profile views.

Features:

* View browsing history
* Track visibility

---

### 4.8 Blocked Profiles

Sub-tabs:

* Blocked by Me
* Blocked by Others

Purpose: Manage blocked profiles.

Features:

* View blocked profiles
* Prevent further interaction

Policy:

* Blocked profiles cannot be unblocked.
* Users must contact support for reconsideration.

---

### 4.9 Reported Profiles

Sub-tabs:

* Reported by Me
* Reported by Others

Purpose: Manage reported profiles.

Features:

* View reported profiles
* Track report status

Policy:

* Reported profiles cannot be unreported.
* Users must contact support for review.

---

### 4.10 Not Interested

Purpose: Manage uninterested profiles.

Features:

* View uninterested profiles
* Mark profile as Interested

---

### 4.11 Details Requested

Sub-tabs:

* Requested by Me
* Requested by Others

Purpose: Manage requests for personal details.

Supported Details:

* Astrology information
* Contact details

Features:

* Send detail requests
* View received requests

Policy:

* Users must be connected before requesting details.

---

## 5. User Actions

Users can perform the following actions from the Activity page:

* Accept or reject requests
* Send and track connection requests
* View favourites and likes
* Block profiles
* Report profiles
* Change interest status
* Request personal details

---

## 6. Restrictions and Policies

1. Blocked profiles cannot be unblocked by users.
2. Reported profiles cannot be unreported by users.
3. Support intervention is required for blocked or reported profile recovery.
4. Detail requests require an active connection.

---

## 7. System Guidelines

* All tabs must display accurate counts.
* Access is restricted to authenticated users.
* Data must be synchronized in real time.
* User privacy must be maintained at all times.
