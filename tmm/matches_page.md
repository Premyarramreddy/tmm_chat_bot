1. Overview

The Matches Page displays user profiles that are relevant to the logged-in user based on preferences, filters, and system recommendations.

Users can:

Search profiles

Apply filters

Browse categorized match tabs

Send connection requests

Add profiles to favorites

View basic profile details

Listen to profile audio introductions

2. Entry Point

Location: Bottom Footer → Matches Icon

Access Condition: User must be logged in

3. Search Functionality

A search bar is available at the top of the Matches Page.

Users can search profiles by:

Country

State

City

Community

Sub-community

First Name

Search results update based on entered value.

4. Location-Based Matching

A Location Icon allows users to enable proximity-based matching.

Behavior

User can set distance range up to 100 km

System shows profiles located within the selected radius

Uses device location permission

5. Filter System

A Filter Icon is available next to the Location icon.

Users can filter profiles using one or more of the following fields:

Age

Height

Marital Status

Religion

Community

Sub-community

Country

State

City

Highest Education Qualification

Profession / Occupation Field

Diet

Disability Status

Profile Managed By

Filter Rules

At least one filter must be selected to apply filters

Click Apply → Filters activated

Click Clear → All filters removed

6. Match Categories (Tabs)

Profiles are grouped into tabs for easier browsing.

Tab Name	Description
Recommended	Profiles suggested by the system based on user data, preferences, and activity
Community	Profiles from the user’s community
Sub-community	Profiles from the user’s sub-community
Popular	Profiles with high engagement (likes, views, activity)
New	Recently registered profiles
Profession	Profiles in the same profession/occupation field
Region	Profiles from the same geographic region

8. Match Percentage

Represents how closely the profile matches the user’s preferences and profile data.

Displayed as a visual percentage badge on each profile.

9. Connection Rules

Users can send connection requests

Chat and interaction features are enabled only after mutual connection

Unconnected users have limited interaction

10. System Behavior Scenarios
Scenario	System Response
No profiles match filters	Show "No profiles found" message
Location permission denied	Show location access prompt
Filters applied	Show only matching profiles
Filters cleared	Reset to default match view
Connection request sent	Show request pending status
Already connected	Show connected state
