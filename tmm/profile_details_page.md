# Profile Details Page – RAG Knowledge Document

## 1. Overview

The Profile Details Page displays complete information about a user’s profile. This page opens when a user selects any profile within the application.

It allows users to view personal, professional, and social information, interact with profiles, and manage connection actions.

## 5. Privacy and Detail Requests

### 5.1 Profile Privacy Settings

Profile details are displayed based on the other user’s privacy preferences.

Users can choose one of the following visibility options in Profile Privacy:

* Show to All
* Only Connected

#### Show to All

If selected, all allowed profile details are visible to everyone.

#### Only Connected

If selected, the following details are hidden and shown as asterisks (*) until a connection is established:

* Photos
* Last name
* Birth date
* Profession 
* Organization
* University / College
* Family details

Once the user connects, these details become visible.

Visibility depends on the user’s selected preference for each field.

---

### 5.2 Restricted Details (Astrology and Contact)

By default, Astrology and Contact Details are hidden and displayed as asterisks (*).

To view these details:

1. User must connect with the profile owner.
2. Click the "Request All" button.
3. A request is sent to the other user.
4. The other user approves selected details.
5. Approved details become visible.

## 8. Activity Page Integration

The following actions are reflected in the Activity Page:

* Add to favourites
* Super like
* Send or accept connection requests
* Block profile
* Report profile
* View profile
* Mark as not interested
* Reject requests

Users can track all these activities through corresponding tabs on the Activity Page.

---

## 9. Restrictions and Policies

1. Contact and astrology details are private by default.
2. Details are visible only after user approval.
3. Blocked and reported profiles cannot be restored by users.
4. All actions are logged in the Activity Module.

---

## 10. System Guidelines

* All profile data must be accurate and up to date.
* Privacy rules must be strictly enforced.
* Only authorized users can view restricted details.
* Real-time synchronization must be maintained.
